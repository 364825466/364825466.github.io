SpringBoot MyBatisPlus

初识MyBatisPlus

MyBatis优势：

  • SQL语句可以自由控制，更灵活，性能较高
  • SQL与代码分离，易于阅读和维护
  • 提供XML标签，支持编写动态SQL

JPA优势：

  • JPA移植性较好（JPQL）
  • 提供了很多CRUD方法，开发效率高
  • 对象化程度更高

对比后MyBatis劣势：

  • 简单CRUD操作还得写SQL语句
  • XML中有大量的SQL要维护
  • MyBatis自身功能优先，但支持Plugin

那么，有没有一种框架，既有MyBatis的种种好处，又拜托了MyBatis的劣势呢？

MyBatis-Plus简介

MP是一个MyBatis的增强工具，只做增强不做改变

地址： https://mp.baomidou.com/

性能介绍：

  • 无侵入，损耗小，强大的CRUD操作
  • 支持Lambda形式调用，支持多种数据库
  • 支持主键自动生成，支持ActiveRecord
  • 支持自定义全局通用操作，支持关键词自动转义
  • 内置代码生成器，内置分页插件，内置性能分析插件
  • 内置全局拦截插件，内置sql注入剥离器

常用注解

User.java 实体类

@Data
@TableName("mp_user")  
// 直接指定对应的数据库表叫什么名 不加的话User默认对应user表 这个代表User对应mp_user表
public class User{

	@TableId	// 指定userId就是对应数据库中的主键
	private Long userId;
	
	// 指定realName在数据库中对应name列  要不然都是默认实体驼峰对应数据库下划线
	@TableField("name")
	private String realName;
	
	private Integer age;
}

排除非表字段的三种方式

场景：实体中某个变量不对应表中任何一个字段，他只是暂时存一些数据或者保存通过某种计算或组装的数据

demo: 给实体类添加一个数据库没有的变量

// 实体类中添加一个数据库没有的字段 用来暂时存一些数据
private String remark;


————————————————————————————————————————————————————————————————————————

// 测试类设置属性值
@Test
public void insert(){
	User user = new User();
	user.setName("小白");
	user.setAge(11);
	
	user.setRemark("我是一个备注信息~");
	int rows = userMapper.insert(user);
}

这是不成功的，因为数据库中没有remark字段，解决办法有三种：

  1. 加关键字transient

    // 实体类中给变量添加关键字
    private transient String remark;

    缺点：加了这个字段不参与序列化了，有序列化的需求用第二种方法

  2. 把它标识为静态变量static

    // 实体类中给变量标识为静态变量
    private static String remark;

    ————————————————————————————————————————

    @Test
    public void insert(){
    	User user = new User();
    	user.setName("小猪");
    	user.setAge(11);
    	
    	// 因为用的static 所以要用类名.
    	User.setRemark("我是一个备注信息~")
    	
    	int rows = userMapper.insert(user);
    }

    注意：

    1. Lombok不会为静态变量声明get set 方法，需要手动生成

2. 因为用的static，所以调用要用类名.方法名

    缺点：这样是全类唯一一份remark，要想每个对象对应一个remark，用第三种

  3. 用@TableField(exist=false)

    // 这个注解exist默认为true，写成false表示不是数据库表中的字段
    @TableField(exist=false)
    private String remark;

条件构造器中condition

使用场景：表示该条件是否加入最后生成的SQL中，如果为true就加入，如果为false就不加入

比如前端给你传值 传过来的sql就加进去，没有传的sql就不写，类似动态sql那种

private void condition(String name,String email){

	QueryWrapper<User> queryWrapper = new QueryWrapper<User>();
	if(StringUtils.isNotEmpty(name)){
		queryWrapper.like("name",name);
	}
	if(StringUtils.isNotEmpty(email)){
		queryWrapper.like("email",email);
	}
	
	List<User> userList = userMapper.selectList(queryWrapper);
	userList.forEach(System.out::println);
	
}

@Test
private void testCondition(){

	String name = '王%';
	String email = '';
	condition(name,email);
}

这样写没有问题，但是太麻烦，可以用带condition的like

private void condition(String name,String email){

	QueryWrapper<User> queryWrapper = new QueryWrapper<User>();
	queryWrapper.like(StringUtils.isNotEmpty(name),"name",name).like(StringUtils.isNotEmpty(email),"email",email);
	
	List<User> userList = userMapper.selectList(queryWrapper);
	userList.forEach(System.out::println);
}

@Test
private void testCondition(){

	String name = '王%';
	String email = '';
	condition(name,email);
}

有时候前端都是要like，不是等值的

修改实体类 加注解

@Data
public class User{
	private Long id;
	// like
	@TableField(condition=SqlCondition.LIKE)
	private String name;
	// 小于
	@TableField(condition="%s&lt;#{%s}")
	private Integer age;
}

这样你写完直接就是like的，然后age就是小于

allEq用法

@Test
public void selectByWrapperAllEq(){

	QueryWrapper<User> queryWrapper = new QueryWrapper<User>();
	Map<String,Object> params = new HashMap<String,Object>();

分页查询

  • MyBatis分页介绍

    不是物理分页，是逻辑分页或内存分页，把符合条件的数据全查出来在内存中，返回你要的那部分

    问题：超大数据情况下耗内存，速度慢

  • MP分页插件实现物理分页

    在SpringBoot环境中新建配置类MybatisPlusConfig

    @Configuration		//标识为配置类的注解
    public class MybatisPlusConfig{

    	@Bean
    	public PaginationInterceptor paginationInterceptor(){
    		return new PaginationInterceptor();
    	}
    }

    测试：

    @Test
    public void selectPage(){
    	QueryWrapper<User> queryWrapper = new QueryWrapper<User>();
    	queryWrapper.ge("age",26)
    	Page<User> page = new Page<User>(1,2);
    	// 	Page<User> page = new Page<User>(1,2，false);  这种是不需要查总记录数

    	
    	/* 写法一
    	IPage<User> iPage = userMapper.selectPage(page,queryWrapper);
    	System.out.println("总页数"+iPage.getPages());
    	System.out.println("总记录"+iPage.getTotal());
    	List<User> userList = iPage.getRecords();
    	*/
    	
    	// 写法2
    	IPage<Map<String,Object>> iPage = userMapper.selectMapsPage(page,queryWrapper);
    	userList.forEach(System.out::println);
    	IPage<User> iPage = userMapper.selectPage(page,queryWrapper);
    	System.out.println("总页数"+iPage.getPages());
    	System.out.println("总记录"+iPage.getTotal());
    	List<Map<String,Object>> userList = iPage.getRecords();
