什么是RESTful

SpringMVC RESTful

RESTful风格概述

什么是RESTful？

REST：Representational State Transfer（表述性状态转移）

REST并不是一种创新技术，它指的是一组架构约束条件和原则

符合REST的约束条件和原则的架构，就称他为RESTful架构

RESTful核心内容

  1. 资源与URI

    表述性状态转移，指的就是资源的表述。那么什么是资源呢？

    在一个系统中，只要一个事务有被引用的必要，那么它就是一个资源

    它可以是一个文本，可以是一首歌曲，也可以是一个服务，总之是一个真实存在的资源

    那么让资源可以被识别，就需要有一个唯一标识，也就是URI，URI就是资源的地址

  2. 资源的表述

    资源在客户端和服务器之间传送

  3. 状态转移

    指资源在客户端发生变迁，然后进入到一个后续的状态，客户端获取资源之后，可能会对资源进行修改，使他跟咱们在服务器保存的状态不一样，那么这个就是状态转移

RESTful架构的特点

  1. 统一了客户端访问资源的接口
  2. url更加简洁，易于理解，便于扩展
  3. 有利于不同系统之间的资源共享

RESTful具体来说就是HTTP协议的四种形式表示四种基本操作

Method CRUD
GET 获取资源
POST 新建资源
PUT 修改资源
DELETE 删除资源

RESTful开发风格

demo：

查询课程：http://localhost:8080/id method=’get’

// 通过id查询课程

Controller:

@GetMapping(value="/getById/{id}")
public ModelAndView getById(@PathVariable(value="id") int id){
	ModelAndView modelAndView = new ModelAndView();
	modelAndView.setViewName("edit");
	modelAndView.addObject("course",courseDAO.getById(id));
	return modelAndView;
}


前端:
<form action="${pageContext.request.contextPath}/getById/${course.id}" method="get">

添加课程：http://localhost:8080/course method=’post’

@PostMapping(value="/add")
public String add(Course course){
	courseDAO.add(course);
}

删除课程：http://localhost:8080/id method=’delete’

@DeleteMapping(value="/delete/{id}"){
	courseDAO.deleteById(id);
	return "redirect:/getAll";
}

修改课程：http://localhost:8080/course method=’put’

@PutMapping(value="/update")
public String update(Course course){
	courseDAO.update(course);
	return "redirect:/getAll";
}
